Features

Everything You Need for CI/CD Failure Analysis

From automated monitoring to AI-powered insights, BugZooka covers the full failure triage pipeline. Scroll sideways →

📡 Always On

Slack Polling Mode

Automatically monitors your CI failure channel at regular intervals. Processes all failure messages without manual intervention.

⚡ Real-Time

Socket Mode

Real-time @mention responses via WebSocket. Trigger on-demand analysis, PR reviews, and nightly inspections by mentioning the bot.

🧠 AI

LLM-Powered Analysis

Uses OpenAI-compatible APIs (Gemini, Llama, DeepSeek) to categorize errors, identify affected components, and explain implications.

📈 Detection

Changepoint Detection

Integrates with orion-mcp for performance regression tracking. Detects changepoints and visualizes PR impact on benchmarks.

📚 RAG

RAG-Augmented Insights

Enriches analysis with Retrieval-Augmented Generation using a local FAISS vector store for context-aware implications.

🔧 Extensible

MCP Server Integration

Extensible tool calling via Model Context Protocol. Supports stdio, SSE, and streamable HTTP transports for external tools.

📊 History

Job Summaries

On-demand failure summaries with rates, trends, and version breakdowns. Just type "summarize 7d" in Slack.

🔍 Analysis

PR Performance Analysis

Gemini + MCP-powered analysis of GitHub pull request performance impact against OpenShift versions.

📦 Deploy

Containerized Deployment

Ready-to-deploy with Docker/Podman, OpenShift, and Kustomize overlays for RAG and chatbot modes.

Get Started

Up and Running in Minutes

Choose your preferred deployment method.

bash 
# Clone and setup
git clone https://github.com/mohit-sheth/BugZooka.git
cd BugZooka
python -m venv venv
source venv/bin/activate

# Install dependencies
make install

# Configure your .env file with Slack tokens and LLM endpoint
cp .env.example .env
# Edit .env with your SLACK_BOT_TOKEN, SLACK_CHANNEL_ID, INFERENCE_URL, etc.

# Run BugZooka
make run ARGS="--enable-inference"

# Or with both polling and socket mode
make run ARGS="--enable-inference --enable-socket-mode"
bash 
# Build image using Podman
podman build -f Dockerfile -t quay.io/YOUR_REPO/bugzooka:latest .

# Push to registry
podman push quay.io/YOUR_REPO/bugzooka:latest

# Run as a container
podman run -d \
  -e ENABLE_INFERENCE=true \
  -e ENABLE_SOCKET_MODE=true \
  -v /path-to/prompt.json:/app/prompt.json:Z \
  -v /path-to/.env:/app/.env:Z \
  quay.io/YOUR_REPO/bugzooka:latest

# Or use Make commands
make podman-build
make podman-run
bash 
# Set required environment variables
export QUAY_CRED='<base64 encoded pull secret>'
export BUGZOOKA_IMAGE='<bugzooka image tag>'
export BUGZOOKA_NAMESPACE='<your namespace>'

# Deploy to OpenShift
make deploy

# Optional: deploy with RAG support
export RAG_IMAGE='<byok rag image tag>'
make deploy

# Optional: deploy with chatbot (socket mode)
export CHATBOT=true
make deploy

# Cleanup
kustomize build ./kustomize | envsubst | oc delete -f -
Commands

Supported Bot Triggers

Interact with BugZooka via Slack messages and @mentions. Hover a row to preview the response.

📊

summarize

Aggregate failure summaries over a time window. Polling

  • summarize 7d Job failure summary for the past 7 days
  • summarize 20m verbose Verbose summary with per-job breakdown
🔍

inspect

Check nightly builds for performance regressions via changepoint detection. Socket

  • @BugZooka inspect <nightly> Check if a nightly build has regressed (15-day lookback)
  • @BugZooka inspect <nightly> for N days Custom lookback period
  • @BugZooka inspect <a> vs <b> Compare two nightly builds
  • @BugZooka inspect <nightly> for config <file> Inspect a specific orion config
📝

analyze pr

Evaluate pull request performance impact against OpenShift baselines. Socket

  • @BugZooka analyze pr: <PR URL>, compare with 4.20 Analyze PR performance against a specific version

MCP Server Configuration

          
Add external tools via mcp_config.json. Supports multiple transports.

          

            
json

            


{
  "mcp_servers": {
    "orion_mcp": {
      "transport": "streamable_http",
      "url": "http://orion-mcp:9001/mcp"
    },
    "github_mcp": {
      "transport": "stdio",
      "command": "docker",
      "args": ["run", "-i", "..."]
    }
  }
}
